**Framework, linguagem e ferramentas**

Utilizei o Angular, com template de SCSS, utilizei também o ngx-pagination para a paginação na página, e para testes usei o padrão do Angular, que é o Jasmine.

**Técnologias X e Y**

Justifique porque você optou pela tecnologia X e não a Y?

Optei pelo Angular pois foi o recomendado pela vaga, não tenho costume de utilizá-lo pois estudo mais React no meu dia a dia, mas já dei olhada por ver diversas vagas que pedem conhecimento no mesmo.

**Princípios de software**

Quais princípios da engenharia de software que você usou?

Bom, para estrutura de pastas dos components e services eu gerei com a CLI do Angular e entao ficou padrão. Para desenvolver as funcionalidades tentei fazer com que o component tivesse apenas o necessário, então a chamada a api estava no service.

**Desafios e problemas**

Conte um pouco sobre os desafios e problemas que você enfrentou e como você resolveu.

Primeiramente tive problemas com o retorno da API, tipei o retorno mas não estava dando certo. Então resolvi refazer a chamada, usando muito console.log para ver o que estava em minhas variáveis, até conseguir pegar os resultados do GET. Após isso foi bem tranquilo, fiz o design e o html com o retorno da API. O maior problema que tive foi com os testes! Infelizmente não consegui realizar todos, mas no service consegui! Tenho pouco conhecimento em testes - seria a próxima coisa que iria estudar - e não consegui encontrar uma explicação clara de como fazer o que queria, que era testar meu component.

**Melhorias e próximas implementações**

O que você entende que pode ser melhorado e como isso pode ser feito?

Posso melhorar o layout, deixando com mais detalhes, posso adicionar uma página para mostrar o repo que o usuário clicar, e deixar minha aplicação com 100% de coverage.
