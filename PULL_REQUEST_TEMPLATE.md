**Framework, linguagem e ferramentas**

Foi utilizado o NestJS como framework principal no backend, dado que ele me oferece uma estrutura robusta para construção de APIs, além de ser um framework que eu já comecei a explorar e com o qual tenho me familiarizado. Para o frontend, optei por usar HTML e CSS. Além disso, utilizei o Postman para testar as rotas da API e o Git para controle de versão.

**Técnologias X e Y**

Inicialmente, foi proposta a utilização do Angular para o frontend, por ser uma tecnologia moderna e que combina bem com o NestJS, mas, devido à curva de aprendizado e ao tempo limitado, optei por implementar apenas o backend com o NestJS e o frontend simples com HTML/CSS. Essa escolha foi motivada pela necessidade de entregar uma funcionalidade básica rapidamente, sem comprometer a integridade do sistema como um todo.

**Princípios de software**

Durante o desenvolvimento, busquei aplicar os seguintes princípios de engenharia de software:

Princípio da simplicidade: Mantive o projeto simples, com uma arquitetura clara e sem sobrecarregar com funcionalidades complexas que poderiam comprometer o prazo.

Princípio de separação de preocupações: No backend, a lógica de negócios foi separada da camada de controle e da camada de dados, permitindo uma manutenção mais fácil e uma estrutura escalável.

**Desafios e problemas**

Um dos maiores desafios foi a integração com o Angular, pois não tenho experiência com o framework e seria meu primeiro contato com o mesmo, o tempo para aprender foi curto então optei a utilização do nestjs apenas.
Outro desafio foi o gerenciamento de estado das tasks no backend, já que eu precisava garantir que as mudanças de status fossem corretamente refletidas na API. Para isso, implementei uma lógica simples de atualização de status, usando um banco de dados em memória.

**Melhorias e próximas implementações**

Uma melhoria que poderia ser feita seria adicionar persistência de dados em um banco de dados real, como o PostgreSQL, para garantir que as tarefas e seus status não sejam perdidos quando o servidor for reiniciado. Além disso, a implementação de autenticação de usuários poderia ser interessante para personalizar a experiência de uso.

Uma outra melhoria seria o frontend, utilizando Angular para uma interface mais interativa, integrando diretamente com o backend de maneira mais dinâmica, talvez com funcionalidades como filtros de tarefas e notificações em tempo real.
